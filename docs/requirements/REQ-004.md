### REQ-004: `matctl course add` / `matctl course remove` — course lifecycle CLI

- **Status:** OPEN
- **Added:** 2026-04-11
- **Completed:** –
- **Verified by:** –
- **Depends on:** REQ-001, REQ-003
- **Description:**

Replace the current "copy an existing course directory, edit `_quarto.yml`,
edit `publish.yml`" ritual with two CLI commands:

```
matctl course add <name> [--title "Human readable title"]
matctl course remove <name>
```

`course add`:

1. Refuses to run if `./projects.yml` is missing (i.e. not in a `material`
   checkout) or if `./<name>/` already exists.
2. Copies `material_core/templates/course/` → `./<name>/` via
   `importlib.resources`.
3. Substitutes `<name>` and the supplied `--title` (defaulting to `<name>`
   title-cased) into `./<name>/_quarto.yml`, `./<name>/index.qmd`, and any
   other templated files. Uses plain string replacement on placeholders the
   template declares (e.g. `{{COURSE_NAME}}`, `{{COURSE_TITLE}}`) — no
   templating engine.
4. Appends `{name: <name>, type: course}` to `projects.yml` using
   `ruamel.yaml` so existing comments and ordering survive.
5. Prints a one-line summary and the suggested next steps (`quarto preview
   <name>`, commit, push).

`course remove`:

1. Refuses to run if `./projects.yml` is missing or if `./<name>/` does not
   exist in the manifest.
2. Removes the entry for `<name>` from `projects.yml`.
3. Deletes `./<name>/` from disk after a confirmation prompt (bypassable with
   `--yes` for scripting). This is a destructive local operation; remote
   content under `material.professorfroehlich.de/<name>/` is **not** touched
   by this command — that cleanup is deliberately manual and documented in
   `administration.md`.

Both commands are idempotent with respect to their own failure modes: partial
failures leave the tree in a state where re-running the command (or the
inverse command) cleans up.

**Dependency additions:**

- `ruamel.yaml` — round-trip YAML edits preserving comments/formatting in
  `projects.yml`. Added to `pyproject.toml` dependencies.

**Template placeholder contract:**

The template currently has hard-coded `digital-und-mikrocomputertechnik`-ish
values in a few places (from REQ-001 extraction). Part of this requirement is
to audit `material_core/templates/course/` and replace any remaining
instances with the declared placeholders (`{{COURSE_NAME}}`,
`{{COURSE_TITLE}}`), then verify a fresh scaffold builds clean.

**Stub cleanup:**

Remove the three `NotImplementedError` stubs from `material_core/cli.py`
(`render`, `deploy`, `new`) as part of this requirement — `render`/`deploy`
were rejected in the REQ-002 promotion discussion, and `new` is superseded by
`course add` / `doc add` (REQ-005).

- **Acceptance criteria:**
  - [ ] `matctl course add demo-course` in a `material` checkout creates
        `./demo-course/` from the template with the name substituted
  - [ ] The same command appends `{name: demo-course, type: course}` to
        `./projects.yml` without corrupting existing comments or entries
  - [ ] A scaffolded `demo-course` builds successfully with
        `quarto render demo-course` (book + PDF) and `quarto render
        demo-course/slides` with no manual post-editing
  - [ ] `matctl course add demo-course` a second time fails cleanly (does not
        overwrite or duplicate the manifest entry)
  - [ ] `matctl course remove demo-course --yes` removes the directory and the
        manifest entry
  - [ ] Running `course remove` on a nonexistent course fails with a clear
        error and changes nothing
  - [ ] `ruamel.yaml` dependency pinned in `pyproject.toml` and installs via
        `pipx install --editable`
  - [ ] `material_core/cli.py` no longer contains `render`, `deploy`, or
        `new` stubs
  - [ ] `material_core/templates/course/` uses declared placeholders; a grep
        for `digital-und-mikrocomputertechnik` inside the template directory
        returns zero hits
  - [ ] `docs/administration.md` and `material/CLAUDE.md` "starting a new
        course" sections updated to show `matctl course add` instead of the
        manual copy ritual

- **Notes:**

- Keep `click` as the CLI framework — `cli.py` already uses it. A `typer`
  migration offers no capability gain for the current command count and costs
  a rewrite of the working `link`/`unlink` commands.
- The `--title` placeholder substitution is intentionally minimal. If richer
  template variables are needed later (course code, term, author), promote to
  a new requirement rather than expanding here.
- `course remove` deliberately does not touch the Cloudflare Worker's KV.
  Tokens issued against a removed course become dead keys. Cleaning them up
  is a token-management concern (deferred, future REQ).
