### REQ-006: `matctl token` — token lifecycle CLI

- **Status:** DONE
- **Added:** 2026-04-11
- **Completed:** 2026-04-11
- **Verified by:** manual issue/list/revoke/show cycle + live browser check against real KV 2026-04-11
- **Depends on:** REQ-001, REQ-004
- **Description:**

Replace `material_core/scripts/manage-tokens.sh` with four `matctl token`
subcommands that call the Cloudflare Workers KV API directly:

```
matctl token issue  <course> <label> [--days 365]
matctl token list   [<course>]
matctl token revoke <token>
matctl token show   <token>
```

All four commands are a direct port of the shell script's functionality.
No KV schema changes — existing tokens continue to work unchanged.

**`token issue <course> <label> [--days 365]`**

1. Generates a 24-character lowercase hex token via `secrets.token_hex(12)`.
2. Writes `{"course": ..., "label": ..., "issued": ..., "expires": ...}` to
   the `tok:<token>` key in the KV namespace.
3. Prints the token, metadata, and the ready-to-paste iLearn URL:
   `https://material.professorfroehlich.de/<course>/?token=<TOKEN>`
   (for `course="*"` the URL uses the bare domain root).

**`token list [<course>]`**

Lists all tokens under the `tok:` prefix, formatted as a table:
`TOKEN | COURSE | LABEL | ISSUED | EXPIRES`. Expired tokens are flagged
`[EXPIRED]`. Pass `<course>` to filter to one course.

**`token revoke <token>`**

Deletes the `tok:<token>` key from KV. Effect is immediate at the Worker.
Already-issued session cookies remain valid (they are HMAC-verified, not
KV-checked). Documents: rotating `COOKIE_SECRET` invalidates all cookies.

**`token show <token>`**

Fetches and pretty-prints the raw JSON metadata for one token.

**Credential loading:**

Read `CF_ACCOUNT_ID`, `CF_API_TOKEN`, and `CF_KV_NAMESPACE_ID` from the
environment, falling back to `scripts/.env` relative to the package root
(same lookup order as the shell script). Fail with a clear error message
if any variable is missing.

**Dependency additions:**

- `httpx` — synchronous HTTP calls to the Cloudflare KV REST API. Added to
  `pyproject.toml` dependencies.

**Cleanup:**

Once all four subcommands are verified, `material_core/scripts/manage-tokens.sh`
and `material_core/scripts/.env.example` may be removed. Update
`docs/administration.md §7` to use `matctl token` throughout.

- **Notes:**

- KV schema is unchanged: key `tok:<token>`, value
  `{"course", "label", "issued", "expires"}`. No migration required.
- `httpx` is synchronous here — no async plumbing needed for four admin
  commands that run sequentially.
- Session cookie validity is independent of KV. Revoking a token does not
  force re-auth for students who have already exchanged it for a cookie.
  Documenting the `COOKIE_SECRET` rotation path in the revoke help text is
  sufficient; no code change needed in the Worker.
- `manage-tokens.sh` should remain until REQ-006 is verified in production,
  so there is no gap in admin capability during development.
