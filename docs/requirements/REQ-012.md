### REQ-012: Fix book chapter numbering and document heading conventions

- **Status:** DONE
- **Added:** 2026-04-26
- **Completed:** 2026-04-26
- **Verified by:** manual render of schutzkonzept-ev-christuskirche (2026-04-26)
- **Depends on:** REQ-005
- **Description:**

  Two adjacent authoring hazards observed in the schutzkonzept project:

  1. **Off-by-one chapter numbering.** `chapters/01-…qmd` is rendered as
     "Kapitel 2" because `index.qmd` is counted as Kapitel 1. The default
     expectation is for `index.qmd` to act as a preface / welcome page
     and not consume a chapter number — but authors should be able to opt
     out and have `index.qmd` become numbered chapter 1 if they prefer.

  2. **Doubled H1.** Authors (humans and assistants) tend to write
     `# Heading {#sec-xxx}` *plus* a YAML `title:`, producing two H1s in
     the rendered HTML. A side-effect: `@sec-…` references break when the
     explicit heading is removed, because the ID was attached to the H1.
     The course template already follows the correct pattern (YAML title
     + H2 sections only) but this is implicit and easily missed.

  Solve both at template + docs level so future scaffolds are correct by
  construction and the author retains explicit control over the preface
  numbering choice.

- **Acceptance criteria:**
  - [x] `templates/course/index.qmd` ships with Quarto's `{.unnumbered}` mechanism applied by default, so `chapters/01-introduction.qmd` is rendered as "Chapter 1" / "Kapitel 1".
  - [x] Removing the `{.unnumbered}` marker from `index.qmd` causes it to render as numbered chapter 1 (verified on a minimal book); when this is done the author is responsible for renumbering chapter file prefixes.
  - [x] `templates/course/index.qmd` carries a short inline comment explaining the unnumbered marker and how to remove it.
  - [x] A scaffolded course with two chapters demonstrates correct numbering on visual inspection (manual render).
  - [x] `templates/course/chapters/01-introduction.qmd` carries an inline comment warning authors not to add a `# Heading` (the YAML `title:` becomes the H1 automatically).
  - [x] `docs/authoring.md` §2 (Document structure → Headings) is updated to state explicitly: in a Quarto Book chapter, the YAML `title:` becomes the H1; do not add `# Heading` as the top-level heading; start content with `## Section`.
  - [x] `docs/authoring.md` §3 (Cross-references) documents that whole-file cross-references via `@sec-…` are not supported by Quarto Book and shows the Markdown-link fallback `[Title](file.qmd)`.
  - [x] `docs/authoring.md` documents the unnumbered-preface convention and how to switch it off.
- **Notes:**

  Source: `material/material-core-input.md` §3 + §4; decision 2026-04-26.

  Implementer should validate the chosen Quarto mechanism (`{.unnumbered}`
  on the chapter title vs YAML-level `number-sections: false` vs the
  `appendices`/parts machinery) against current Quarto docs before
  committing — §4 records that `number-sections: false` on the index file
  alone had no effect, and dropping `index.qmd` from `book.chapters`
  broke rendering. Both are dead ends.

  This requirement applies to the course template and to any
  multi-chapter book template introduced by REQ-013.
