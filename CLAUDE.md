# material-core ‚Äî Claude Code Guide

## What this repo is

Engineering tooling for the [`material`](https://github.com/pfroehlich/material)
content repository: the `matctl` Python CLI, the brand registry (`brands/`,
`shared/`), the Cloudflare access Worker, and the course scaffolder template.

**User-facing documentation** (authoring guide and administration manual) lives
in `material/material/` as a rendered Quarto book ‚Äî see `material/material/chapters/`.
Engineering internals (plans, requirements) remain here under `docs/`.

`material-core` is consumed by `material` via a pinned git tag ‚Äî no PyPI
release. Local development uses an editable `pipx` install so SCSS/brand edits
are visible to `quarto preview` in `material` without any sync step.

Scope boundary: **no lecture content lives here.** Content ‚Äî chapters, slides,
assets for specific courses ‚Äî belongs in `material/`.

## matctl CLI

- `matctl link [--force]` ‚Äî at the repo root: fan out per-project symlinks
  (`_brand.yml`, `brand.scss`, `brand-assets/`, `shared/`) for every course/doc
  entry, plus the root-level `shared/` symlink. Inside a project directory: wire
  just that project. Broken symlinks (absolute paths committed by mistake) are
  always replaced even without `--force`. With `pipx install --editable`,
  symlinks resolve into the live source tree, so SCSS edits are immediately
  visible to `quarto preview`.
- `matctl unlink` ‚Äî remove the per-project brand symlinks (and `shared/` at root).
- `matctl group add <name> --title "..."` ‚Äî register a new group in
  `projects.yml`. No directory created. Groups must exist before any course or
  doc can reference them via `--group`. Regenerates `<group>/index.html`
  (no-op on disk ‚Äî no children yet).
- `matctl group remove <name> [--yes]` ‚Äî remove a group entry (fails if any
  course/doc still references it). No landing-page regeneration ‚Äî group is
  already empty by precondition.
- `matctl group modify <name> --title "..."` ‚Äî update the group's title in the
  manifest and regenerate `<group>/index.html`.
- `matctl project add <name> --structure [chapters|single] --slides|--no-slides --brand <name> --lang [de|en] [--title "..."] [--subtitle "..."] [--group <name>]`
  ‚Äî scaffold a new project from orthogonal template fragments and register it in
  `projects.yml`. The three orthogonal axes produce all eight combinations:
  - `--structure chapters` ‚Üí multi-chapter Quarto book (replaces `course add`)
  - `--structure single` ‚Üí single-file document (replaces `doc add`)
  - `--slides/--no-slides` ‚Üí include a `slides/` subdirectory with RevealJS deck
  - `--brand` ‚Üí visual identity (generic, thd, pf); required
  - `--lang de|en` ‚Üí crossref label language; required
  `--subtitle` only applies to `--structure chapters`. `--group` requires the group
  to exist; deploys under `<group>/<name>/`. Regenerates `<group>/index.html`.
- `matctl project remove <name> [--yes]` ‚Äî remove the project directory and
  manifest entry (remote content and KV tokens must be cleaned up manually).
  Regenerates `<group>/index.html` for the removed entry's group (if any).
- `matctl project modify <name> [--title "..."] [--group <name>] [--brand <name>] [--slides|--no-slides] [--lang de|en]`
  ‚Äî update a project's metadata. `--title` is written through to `_quarto.yml:book.title`
  (chapters) or `index.qmd` front matter (single). `--brand` rewires per-project
  symlinks and updates `_quarto.yml` favicon/sidebar.logo in place. `--slides`
  (false‚Üítrue) adds the slides overlay; `--no-slides` rejected when `slides/` has content.
  `--structure` is always rejected ‚Äî create a new project and move content by hand.
  Regenerates landing pages for all affected groups.

  v0.6.0 ‚Üí v0.7.0 migration:
  ```
  matctl course add  ‚Üí matctl project add --structure chapters --slides     --brand thd --lang de
  matctl doc    add  ‚Üí matctl project add --structure single   --no-slides  --brand thd --lang de
  matctl course/doc remove ‚Üí matctl project remove
  matctl course/doc modify ‚Üí matctl project modify
  ```
  Legacy `type: course` / `type: doc` entries in `projects.yml` are migrated to
  `type: project` + `structure` + `slides` + `brand` + `lang` automatically on
  the next `project modify` or `project add` that triggers a manifest save.
- `matctl token issue <course> <label> [--days 365]` ‚Äî generate a token, write
  it to Cloudflare KV, and print the ready-to-paste iLearn URL.
- `matctl token list [<course>]` ‚Äî table of all tokens (or filtered to one
  course); expired tokens flagged `[EXPIRED]`.
- `matctl token revoke <token>` ‚Äî delete the KV entry; effect is immediate at
  the Worker.
- `matctl token show <token>` ‚Äî pretty-print the raw KV JSON for one token.

## How this repo is consumed

CI (pinned):

```bash
pipx install "git+https://github.com/pfroehlich/material-core@v0.7.0"
matctl link
quarto render <course>
```

Local development (editable):

```bash
pipx install --editable /home/peter/projects/material-core
cd /home/peter/material     # formerly vorlesungen
matctl link
quarto preview <course>     # picks up live shared/base.scss edits
```

## Requirements tracking

New requirements, plans, and status changes live in `docs/requirements/` and
`docs/plans/` of *this* repo ‚Äî not in `material/`. Follow the workflow in
[`docs/requirements/Requirements_Management.md`](docs/requirements/Requirements_Management.md);
read [`REQUIREMENTS_INDEX.md`](docs/requirements/REQUIREMENTS_INDEX.md) at
session start.

## Release flow

1. Commit to `main`.
2. Tag `vX.Y.Z`, push the tag.
3. Update the pinned version in `material/.github/workflows/publish.yml`.

Semver: breaking CLI or brand-contract changes bump major; new commands or
assets bump minor; fixes bump patch.
